# V0.2 Roadmap

Priority-ordered plan for the next release. Ordered by blocker status: packages
that unblock others come first. Each item maps to a Pingora crate and lists
the v0.1 gaps that must be filled.

---

## Phase 1 — Unblockers (packages with zero or near-zero deps)

### 1. `zigora_http` — ResponseHeader + CaseMap

Pingora crates: `pingora-http`

v0.1 gap: `RequestHeader` exists but `ResponseHeader` and `CaseMap` are missing.
Every v0.2 proxy improvement (retry, error responses, cache headers) needs these.

- [ ] `ResponseHeader.parse_from(buf) → ResponseHeader` — status_code, version, headers, reason_phrase
- [ ] `ResponseHeader.header_to_wire(buf) → void` — serialize back to HTTP/1.1
- [ ] `CaseMap` — case-preserving header name map. `append_header(name, value)` stores original case alongside lowercased lookup
- [ ] `header_to_h1_wire` — serialise HMap with preserved case
- [ ] `HttpTask` enum — `Header | Body | Trailer | Done | Failed`. Used by the proxy pipe in phase 4

### 2. `zigora_limits` — Rate limiter + Inflight counter

Pingora crate: `pingora-limits`. Standalone (no deps inside the framework).

- [ ] `Estimator` — Count-Min sketch: `hashes × slots` grid of atomic counters. `incr(key, value) → isize` (min across hashes)
- [ ] `Rate` — sliding window estimator: two `Estimator` slots (red/blue), toggle every `interval`. `rate(key) → f64` (avg per second)
- [ ] `Inflight` — `incr(key, value) → (Guard, isize)`. Guard auto-decrements on drop

### 3. `zigora_lru` — Sharded weighted LRU

Pingora crate: `pingora-lru`. Standalone. Used by eviction in `zigora_cache`.

- [ ] `Lru<T, N: usize>` — N-way sharded LRU with per-shard `RwLock`
- [ ] `admit(key, data, weight) → usize` — returns shard index
- [ ] `evict_to_limit() → Vec<(T, usize)>` — until under weight limit
- [ ] `promote(key)`, `remove(key)`, `peek(key)`, `len()`, `weight()`
- [ ] Internal: `LruUnit<T>` — `HashMap<u64, LruNode>` + linked-list order

### 4. `zigora_ketama` — Consistent hash ring

Pingora crate: `pingora-ketama`. Standalone. Used by `zigora_lb`'s `Consistent` selector.

- [ ] `Bucket { node: SocketAddr, weight: u32 }`
- [ ] `Continuum` — ring of `Point` entries sorted by CRC32 hash
- [ ] `Continuum.new(buckets) → Continuum` — D1V4: 160 points per weight
- [ ] `node_idx(hash_key) → usize` — binary search on the ring
- [ ] `NodeIterator` — wraps `get_addr(idx)` for iteration

### 5. `zigora_tinyufo` — S3-FIFO + TinyLFU cache algorithm

Pingora crate: `tinyufo`. Standalone. Used by `zigora_memory_cache` (which `zigora_cache` uses).

- [ ] `TinyUfo<K, T>` — `new(total_weight_limit, estimated_size)`
- [ ] `get(key) → Option<T>` — TinyLFU admission check
- [ ] `put(key, data, weight) → Vec<KV<T>>` — evictions returned to caller
- [ ] Two FIFO queues: small (10% of weight) + main. Ghost TinyLFU estimator
- [ ] Internal: `SegQueue<Key>` (lock-free), `Bucket<T> { uses, queue, weight, data }`

## Phase 2 — Composite packages (depend on phase 1)

### 6. `zigora_pool` — Connection pool (L4 connection reuse)

Pingora crate: `pingora-pool`. Depends on `zigora_lru` (for eviction ordering).

- [ ] `ConnectionPool<S>` — `HashMap<GroupKey, PoolNode>` + `Lru<ID, ConnectionMeta>`
- [ ] `PoolNode<T>` — `Mutex<HashMap<ID, T>>` + lock-free hot queue
- [ ] `get(key) → Option<S>` — try hot queue, then mutex
- [ ] `put(meta, connection) → (Notify, Receiver)` — store for reuse
- [ ] `idle_poll` / `idle_timeout` — background connection sweeper

### 7. `zigora_memory_cache` — TinyUFO-backed in-memory cache

Pingora crate: `pingora-memory-cache`. Depends on `zigora_tinyufo`.

- [ ] `MemoryCache<K: Hash, T: Clone>` — wraps `TinyUfo<u64, Node<T>>`
- [ ] `get(key) → (Option<T>, CacheStatus)` — `Hit | Miss | Expired | LockHit | Stale(dur)`
- [ ] `put(key, value, ttl: ?Duration)` — skips if ttl == 0
- [ ] TTL lazy expiration on `get`, no background purge
- [ ] Weight always 1 per entry

### 8. `zigora_lb` — Load balancer with backend selection

Pingora crate: `pingora-load-balancing`. Depends on `zigora_ketama`, `zigora_core`.

- [ ] `Backend { addr: SocketAddr, weight: usize }` — `new(addr_str) → Result`
- [ ] `Backends` — discovery + health check + `ArcSwap<BTreeSet<Backend>>`
- [ ] `LoadBalancer<S: BackendSelection>` — `select(key, max_iter) → Option<Backend>`
- [ ] `BackendSelection` trait — `build(backends) → Self`, `iter(key) → Iter`
- [ ] Four selectors: `RoundRobin`, `Random`, `FNVHash`, `Consistent` (uses `zigora_ketama`)
- [ ] `Weighted<H>` — duplicates backend indices by weight
- [ ] `HealthCheck` trait — `check(backend) → Result`. `TcpHealthCheck` built-in

### 9. `zigora_cache` — HTTP caching state machine

Pingora crate: `pingora-cache`. Depends on `zigora_http`, `zigora_lru`, `zigora_memory_cache`.

- [ ] `HttpCache` — `phase: CachePhase`. `enable(storage, eviction, predictor, lock)`
- [ ] `CachePhase` enum — `Disabled | CacheKey | Hit | Miss | Stale | Expired | Revalidated`
- [ ] `cache_lookup()`, `cache_found(meta, handler, status)`, `cache_miss()`
- [ ] `Storage` trait — `lookup(key) → (CacheMeta, HitHandler)`, `get_miss_handler(key, meta) → MissHandler`
- [ ] `HitHandler` trait — `read_body() → ?Bytes`, `finish()`
- [ ] `MissHandler` trait — `write_body(data, eof)`, `finish() → MissFinishType`
- [ ] `CacheMeta` — `fresh_until`, `created`, `updated`, `variance`, `header: ResponseHeader`
- [ ] `EvictionManager` trait — `admit(item, size, fresh_until) → Vec<CompactCacheKey>`

## Phase 3 — Proxy upgrades (require phase 1 + 2)

### 10. `zigora_proxy` — Full ProxyHttp chain + Session + Retry

Pingora crate: `pingora-proxy`. Adds ~25 callbacks to the current 3.

- [ ] `Session` struct — `downstream_session`, `request: RequestHeader`, `response: ?ResponseHeader`, `cache: ?HttpCache`
- [ ] `early_request_filter()`, `request_filter()`, `request_body_filter()`
- [ ] `proxy_upstream_filter()` — boolean gate before upstream
- [ ] `upstream_request_filter(session, upstream_request)` — modify request for backend
- [ ] `upstream_response_filter(session, upstream_response)` — inspect upstream response
- [ ] `upstream_response_body_filter()` — streaming body pass-through
- [ ] `response_filter(session, response)` — modify response for client
- [ ] `response_body_filter()` — streaming pass-through from upstream to client
- [ ] `response_cache_filter()` → `RespCacheable` — cache decision
- [ ] `logging(session, error, ctx)` — structured log line
- [ ] `fail_to_connect(session, peer, ctx, error) → Box<Error>` — error enrichment
- [ ] `fail_to_proxy(session, error, ctx) → FailToProxy` — HTTP error response
- [ ] `error_while_proxy(peer, session, error, ctx) → Box<Error>`
- [ ] `connected_to_upstream(session, reuse, peer, fd, digest, ctx)` — notify
- [ ] Retry loop — `upstream_peer → connect → proxy_to_upstream → on error: set_retry + next peer`
- [ ] `upstream_peer()` — now driven by `LoadBalancer.select()` instead of fixed `HttpPeer`

### 11. `zigora_tls` — TLS accept/connect

Pingora crates: `pingora-openssl` / `pingora-boringssl` / `pingora-rustls` / `pingora-s2n`.

- [ ] `tls::accept(stream, config) → TlsStream` — server-side TLS
- [ ] `tls::connect(stream, peer, config) → TlsStream` — upstream TLS
- [ ] One TLS backend (BoringSSL or rustls, mutually exclusive)
- [ ] `ConnectorOptions.ca_file`, `cert_key_file`, `debug_ssl_keylog`
- [ ] ALPN negotiation for H2

## Phase 4 — Server upgrades

### 12. `zigora_core` — Shutdown signals, FD transfer, keepalive

Pingora crate: `pingora-core`. Upgrades to Server + Service.

- [ ] `sigaction(SIGINT/SIGTERM/SIGQUIT)` → `shutdown_flag.store(true)`
- [ ] `ShutdownWatch` — receiver that accept loop polls per iteration
- [ ] `ExecutionPhase` state machine — `Running → GracefulTerminate → ShutdownStarted → Terminated`
- [ ] `HTTP/2 server options` — `H2Options`, ALPN negotiation
- [ ] `ConnectionPool` integration — `Connector.get_http_session(peer)` uses pool
- [ ] Keepalive — `process_new` returns `Some(stream)` → re-accept on same connection
- [ ] `add_tls("addr", cert_path, key_path)` on `Listeners`
- [ ] Multi-listener: `listeners_per_fd > 1` → multiple accept tasks per listener

### 13. `zigora_metrics` — Prometheus endpoint + Admin status

Pingora crate: `pingora-prometheus`.

- [ ] `GET /metrics` HTTP endpoint
- [ ] Atomic counters: connections accepted, bytes proxied, errors by type
- [ ] Per-upstream latency histograms (optional, stdlib rolling bucket)

---

## Phase 5 — Deferred indefinitely

- `zigora-header-serde` — zstd dictionary-based header compression. Pingora uses it for bandwidth optimisation; low priority for Zigora since the proxy pipeline is the focus
- `zigora-runtime` — no port needed (Zigora uses `std.Io` directly)
- `zigora-timeout` — not needed (`std.Io.Timeout` covers it)

## v0.2 acceptance test (adds to v0.1's)

```bash
# Rejects to health check
# multiple backends with round-robin
zig build run -- --backend 127.0.0.1:9000 --backend 127.0.0.1:9001

# TLS upstream
curl --cacert cert.pem https://127.0.0.1:8443/

# Cache hit
curl -v http://127.0.0.1:8080/  # first: miss → 200
curl -v http://127.0.0.1:8080/  # second: hit → 200 (age header)
```
